Frequently Asked Questions

1. What is HTML?

HTML stands for Hypertext Markup Language. It’s the standard language used to create web pages. With HTML, you define the structure and content of a webpage using tags and elements.

2. Why do we use HTML?

We use HTML to create structured documents that can be displayed in web browsers. It allows us to define headings, paragraphs, images, links, forms, and other elements necessary for building web pages.

3.What is <!DOCTYPE> in HTML?

<!DOCTYPE> declaration specifies the document type and version of HTML being used. It helps browsers understand how to interpret the HTML code.

5.What is a tag?

A tag is an HTML element enclosed in angle brackets (e.g., <p>, <a>, <img>). Tags define the structure and content of a webpage.

6.Examples of self-closing tags

Some self-closing tags include <img>, <br>, and <input>.

7.What is an element?

An element consists of an opening tag, content, and a closing tag. eg:( <p>Some text</p>)

8.How many types of elements are there in html?

There are two main types: inline elements (e.g., span, a) and block elements (e.g., div, p).

9.What is difference between the inline vs block elements?

Inline elements don’t create line breaks and flow within the text, while block elements start on a new line and create a block-level structure.

10.What is difference between the semantic elements and non-semantic elements?

Semantic elements clearly define their meaning in a human- and machine-readable way (e.g., article, header, footer). Non-semantic elements do not provide information about their content (e.g., div, span).

11.What are href vs src attributes?

href is used for hyperlinks (e.g., linking to other pages), while src specifies the source of an external resource (e.g., an image).

12.What are the features of html 5?

26.What is git vs github?

Git is a version control system, while GitHub is a platform for hosting Git repositories.

27.What is the workflow of github?

It involves creating a repository, committing changes, pushing to GitHub, and collaborating with others.

28.What command do we need to use to move the changes to staged changes?

To move changes from the working directory to the staging area (index), you use the git add command. For example: git add filename.txt

29.How can you stage all files at a time?

To stage all modified files at once, use: git add .

30.What is the git push command and use of it?

The git push command is used to upload local commits to a remote repository (like GitHub). It pushes your changes from your local branch to the corresponding branch on the remote server.

31.What is git commit command usage?

The git commit command creates a new commit with the changes you’ve staged (using git add). You provide a commit message to describe the changes made. Example: git commit -m "Added new feature"

32.What is the use of the git init command?

git init initializes a new Git repository in the current directory. It sets up the necessary Git files and directories.
